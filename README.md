# Курсовой проект «Змейка» (C++ / SFML 3.1)

## Что умеет программа

- Главное меню: «Новая игра», «Рекорды», «Выход».
- Игровое поле с сеткой, рамкой и HUD (счёт, длина, скорость).
- Змейка, которая движется по сетке стрелками или WASD.
- Случайно появляющиеся фрукты — за каждый съеденный фрукт начисляется 10 очков и змейка удлиняется.
- Скорость змейки увеличивается по мере роста.
- Обработка столкновений со стеной и с собственным телом → «Game Over».
- Пауза по пробелу.
- Таблица рекордов (топ-10), сохраняется в файле `scores.txt` рядом с исполняемым файлом.
- Ввод имени игрока после окончания игры.
- Информация о текущем состоянии игры дублируется в заголовке окна.

## Шаг 1. Устанавливаем Visual Studio 2026

Если у вас уже установлена **Visual Studio 2026** с поддержкой C++ и компонентами CMake — пропустите этот шаг.

1. Скачайте установщик с официального сайта: [https://visualstudio.microsoft.com/ru/vs/](https://visualstudio.microsoft.com/ru/vs/) (выберите редакцию Community 2026).
2. Запустите установщик. В окне выбора рабочих нагрузок отметьте галочкой:
   - **«Разработка классических приложений на C++»** (Desktop development with C++).
3. Справа, в панели «Сведения об установке», убедитесь, что выбраны следующие пункты (в разделе «Дополнительно»):
   - **«Средства сборки MSVC для x64/x86 (последний...)»** — это и есть компилятор C++ для VS 2026 (в версии 18.0+ это внутренне тулсет **v145** / MSVC 14.50, но в установщике он отображается просто как «последний»). Обычно уже отмечен по умолчанию — просто проверьте галочку.
   - **«Средства CMake C++ для Windows»** — включает CMake 4.1+ и Ninja. Это **обязательный** пункт — без него проект не соберётся.
   - **«Диспетчер пакетов vcpkg»** — желательно включить. Это системный vcpkg от Microsoft — он не заменяет ручную установку vcpkg (Шаг 2), но упрощает жизнь.
   - **«Windows 11 SDK»** (свежий, например 10.0.26100.0 или новее).

   Пункты «MSVC версии 143», «MSVC версии 142», «MSVC версии 141» **включать не нужно** — это старые тулсеты из VS 2022/2019/2017, которые ставятся side-by-side для сборки старых проектов. Наш проект их не использует.

4. Нажмите «Изменить» / «Установить» и дождитесь окончания установки (это займёт 30–60 минут при чистой установке и пару минут, если вы просто добавляете компоненты к уже установленной VS).

> **Важно:** Visual Studio 2026 использует новый генератор CMake `Visual Studio 18 2026` и внутренне тулсет v145 (MSVC 14.50). В `CMakePresets.json` мы **не указываем тулсет явно** — тогда CMake возьмёт тот, что встроен в вашу VS 2026 по умолчанию. Менять в пресетах ничего не нужно.

## Шаг 2. Устанавливаем vcpkg

**vcpkg** — это менеджер C++ библиотек от Microsoft. С его помощью мы установим SFML.

1. Откройте **PowerShell** (нажмите Win + X → «Терминал» или «Windows PowerShell»).
2. Перейдите в папку, где будет лежать vcpkg. Рекомендую `C:\dev` или `C:\tools`. Если такой папки нет — создайте:
   ```powershell
   mkdir C:\dev
   cd C:\dev
   ```
3. Клонируйте репозиторий vcpkg:
   ```powershell
   git clone https://github.com/microsoft/vcpkg.git
   ```
   Будет создана папка `C:\dev\vcpkg`.
4. Перейдите в неё и запустите скрипт инициализации:
   ```powershell
   cd C:\dev\vcpkg
   .\bootstrap-vcpkg.bat
   ```
   Через 1–2 минуты появится файл `vcpkg.exe`.
5. Интегрируйте vcpkg с Visual Studio (выполнить **один раз** на всём компьютере):
   ```powershell
   .\vcpkg.exe integrate install
   ```
   В ответ должно появиться сообщение «Applied user-wide integration for this vcpkg root».

> **Важно:** vcpkg сам определит, что у вас Visual Studio 2026, и при компиляции пакетов будет использовать тулсет **v145**. Никакой дополнительной настройки `VCPKG_PLATFORM_TOOLSET` не требуется.

## Шаг 3. Устанавливаем SFML 3.1 через vcpkg (manifest-режим)

В **Visual Studio 2026** вместе со студией ставится встроенный vcpkg по пути `C:\Program Files\Microsoft Visual Studio\18\Community\VC\vcpkg`. Наш `CMakePresets.json` **уже явно указывает этот путь** в поле `CMAKE_TOOLCHAIN_FILE`, поэтому CMake без проблем найдёт toolchain.

Этот встроенный vcpkg работает только в так называемом **manifest-режиме** — зависимости проекта описываются в файле `vcpkg.json` рядом с `CMakeLists.txt`, и vcpkg ставит ровно то, что в нём перечислено, **автоматически** при первой настройке CMake. Никаких команд `vcpkg install ...` вручную выполнять не нужно.

> **Если у вас Visual Studio 2026 установлена не в `C:\Program Files\Microsoft Visual Studio\18\Community`**, а в другом месте (например, Professional / Enterprise / другой диск) — откройте `CMakePresets.json` в блокноте и исправьте путь в поле `CMAKE_TOOLCHAIN_FILE` на соответствующий вашему. Проверить, где лежит встроенный vcpkg, можно командой в Developer PowerShell:
> ```powershell
> where.exe vcpkg
> ```

В нашем проекте такой файл `vcpkg.json` **уже есть** в корне (рядом с `CMakeLists.txt`):

```json
{
    "name": "snake",
    "version-string": "1.0.0",
    "dependencies": [
        { "name": "sfml", "version>=": "3.0.0" }
    ],
    "builtin-baseline": "aa40adda5352e87655b8583cfb2451d5e9e276fd"
}
```

Что здесь к чему:
- `dependencies` — список нужных библиотек. Нам нужна только SFML 3.x.
- `builtin-baseline` — конкретный коммит реестра vcpkg, который фиксирует версии всех пакетов. Это гарантирует, что у вас и у проверяющего проект соберётся с одинаковой версией SFML.

**Что произойдёт автоматически:**

Когда вы откроете папку проекта в Visual Studio 2026 (Шаг 5), CMake вместе со встроенным vcpkg прочитает `vcpkg.json` и **сам** соберёт SFML 3.x под x64-windows. Это произойдёт прямо во время первой настройки и займёт от 10 до 30 минут — vcpkg компилирует SFML и её зависимости из исходников. В нижнем окне «Вывод» вы увидите строки вида «Installing 1/8 sfml...».

> **Если вы используете ручной vcpkg из `C:\dev\vcpkg`** (для других проектов или из старых инструкций) — для этого проекта он **не нужен**. Встроенный vcpkg от Visual Studio 2026 справится сам. Удалять ручной vcpkg не обязательно, он не мешает.

## Шаг 4. (Опционально) Переменная VCPKG_ROOT

**Этот шаг можно пропустить**, если вы пользуетесь только встроенным vcpkg от Visual Studio 2026 (что мы и рекомендуем). Visual Studio 2026 автоматически подцепит свой vcpkg, никакая переменная окружения для этого не нужна.

Если же вы хотите принудительно использовать **ручной** vcpkg (например, `C:\dev\vcpkg`, в котором у вас уже стоит SFML) — задайте переменную окружения:

```powershell
[Environment]::SetEnvironmentVariable("VCPKG_ROOT", "C:\dev\vcpkg", "User")
```

После этого **закройте все окна PowerShell и Visual Studio** и откройте заново. Проверка:
```powershell
echo $env:VCPKG_ROOT
```

`CMakePresets.json` написан так, что использует `VCPKG_ROOT`, если он задан, — иначе берёт встроенный vcpkg от VS 2026.

## Шаг 5. Открываем проект в Visual Studio 2026

> **КРИТИЧЕСКИ ВАЖНО:** путь к папке проекта **не должен содержать кириллицу, пробелы и спецсимволы**. vcpkg при сборке SFML из исходников спотыкается о пути вида `D:\курсовая\Snake` и даёт неочевидные ошибки вроде «Could not find a package configuration file provided by SFML». Правильные пути: `C:\projects\Snake`, `D:\dev\Snake`, `D:\study\Snake`. **Неправильные**: `D:\курсовая\Snake`, `C:\Мои проекты\Snake`.

1. Распакуйте архив с проектом в удобное место с английским путём, например `C:\projects\Snake` или `D:\dev\Snake`. После распаковки внутри должны лежать файлы `CMakeLists.txt`, `CMakePresets.json`, `vcpkg.json`, папки `src/`, `include/`, `assets/`.
2. Запустите **Visual Studio 2026**.
3. На стартовом экране выберите **«Открыть локальную папку»** (Open a local folder).
   *Если Visual Studio уже запущена:* меню **Файл → Открыть → Папка…**
4. Выберите папку `C:\projects\Snake` (ту, в которой лежит `CMakeLists.txt`) и нажмите «Выбор папки».
5. Visual Studio автоматически обнаружит `CMakeLists.txt` и `CMakePresets.json`. Внизу в строке состояния вы увидите процесс «Создание кеша CMake» / «CMake generation» — это нормально, идёт первичная настройка проекта (может занять 1–3 минуты).
6. В верхней панели Visual Studio есть выпадающий список конфигурации (рядом со значком запуска). Убедитесь, что выбрано:
   - **Конфигурация:** `vs2026-x64` (это **рекомендуемый** пресет — использует встроенный vcpkg от VS 2026)
   - **Целевой объект запуска:** `Snake.exe`

   В списке также будет пресет `vs2026-x64-manual-vcpkg` — это вариант для ручного vcpkg из `VCPKG_ROOT`. Не выбирайте его, если не уверены, что он вам нужен.

   Если списка конфигураций нет или он пустой — значит у вас не установлен компонент «Инструменты C++ CMake для Windows» или «Диспетчер пакетов vcpkg». Вернитесь к шагам 1, затем перезапустите Visual Studio.

## Шаг 6. Собираем проект

1. В меню Visual Studio выберите **Сборка → Собрать всё** (Build → Build All), либо нажмите **Ctrl + Shift + B**.
2. **При первой сборке** в окне «Вывод» вы увидите строки вида «Installing 1/8 sfml...» — это vcpkg в manifest-режиме скачивает и компилирует SFML и её зависимости. Этот этап займёт 10–30 минут — это нормально и бывает только один раз. Дождитесь окончания компиляции. Внизу в окне «Вывод» должно появиться:
   ```
   Сборка завершена.
   ```
3. Готовый `Snake.exe` будет лежать по пути:
   ```
   C:\projects\Snake\build\vs2026-x64\bin\Debug\Snake.exe
   ```
   (или `\Release\Snake.exe` — зависит от выбранной конфигурации). Рядом с ним автоматически окажется папка `assets/` со шрифтом — это делает CMake после сборки.

## Шаг 7. Запускаем игру

**Из Visual Studio:** просто нажмите зелёный треугольник «Запустить» в верхней панели либо клавишу **F5** (запуск с отладкой) или **Ctrl + F5** (без отладки).

**Из проводника:** перейдите в папку `build\vs2026-x64\bin\Debug\` (или `\Release\`) и дважды кликните по `Snake.exe`.

> **Важно:** не запускайте `Snake.exe` отдельно из любой другой папки — программа ищет файл шрифта по относительному пути `assets/DejaVuSans.ttf`. Без него запуск завершится с сообщением об ошибке.

## Структура проекта (что за какие файлы отвечает)

```
Snake/
├── CMakeLists.txt          — главный файл сборки CMake
├── CMakePresets.json       — описание пресета "vs2026-x64"
├── README.md               — этот файл
├── assets/
│   └── DejaVuSans.ttf      — шрифт с поддержкой кириллицы
├── include/                — заголовочные файлы (объявления классов)
│   ├── Config.hpp          — все настраиваемые константы игры
│   ├── Field.hpp           — игровое поле
│   ├── Fruit.hpp           — фрукт
│   ├── Game.hpp            — главный класс игры
│   ├── Menu.hpp            — меню и экран рекордов
│   ├── ScoreManager.hpp    — таблица рекордов (загрузка/сохранение)
│   └── Snake.hpp           — змейка
└── src/                    — исходные файлы (реализации)
    ├── Field.cpp
    ├── Fruit.cpp
    ├── Game.cpp            — игровой цикл, обработка ввода, отрисовка
    ├── main.cpp            — точка входа в программу
    ├── Menu.cpp
    ├── ScoreManager.cpp
    └── Snake.cpp
```

**Краткое назначение каждого модуля:**

- **Config.hpp** — единственное место, где нужно менять числовые параметры игры (размер клетки, размер поля, начальная скорость, очки за фрукт и т. д.).
- **Snake** — модель змейки: список клеток (на основе `std::deque`), движение, проверка самопересечения.
- **Fruit** — фрукт: появляется в случайной свободной клетке, отрисовывается кругом.
- **Field** — отрисовка фона, сетки и рамки игрового поля.
- **ScoreManager** — чтение и запись файла `scores.txt`, поддержание топа-10.
- **Menu** — главное меню и экран таблицы рекордов.
- **Game** — основной класс. Содержит конечный автомат состояний (Меню → Игра → Пауза → Game Over → Ввод имени → Рекорды) и игровой цикл с фиксированным шагом времени.
- **main.cpp** — создаёт объект `Game` и запускает его. Внутри `try/catch` — если что-то сломалось, ошибка будет распечатана в консоль.

## Настройка игры (изменение констант)

Все параметры, которые имеет смысл менять, собраны в одном файле — `include/Config.hpp`, в пространстве имён `cfg::`. Например:

```cpp
constexpr unsigned int CELL_SIZE = 24;            // размер клетки в пикселях
constexpr unsigned int COLS = 25;                 // ширина поля в клетках
constexpr unsigned int ROWS = 20;                 // высота поля в клетках
constexpr unsigned int SNAKE_START_LENGTH = 4;    // начальная длина змейки
constexpr float INITIAL_STEPS_PER_SECOND = 8.0f;  // начальная скорость (шагов в секунду)
constexpr float MAX_STEPS_PER_SECOND = 22.0f;     // максимальная скорость
constexpr float SPEED_INCREMENT = 0.4f;           // прирост скорости за фрукт
constexpr int FRUIT_SCORE = 10;                   // очки за один фрукт
constexpr std::size_t MAX_SCORES = 10;            // сколько рекордов хранить
```

После изменения любой константы нужно **пересобрать проект** (Ctrl + Shift + B).

## Возможные проблемы и их решения

### «CMake не может найти SFML» (Could not find a package configuration file provided by "SFML")
Это самая частая ошибка при первом запуске. Проверяйте по порядку:

1. **Путь к папке проекта содержит кириллицу или пробелы?** Если да (например, `D:\курсовая\Snake`) — vcpkg не сможет собрать SFML. **Перенесите папку** в путь без кириллицы и пробелов, например `C:\projects\Snake` или `D:\study\Snake`. После переноса удалите папки `build/` и `.vs/`, если они остались, и откройте проект заново.

2. **Файл `vcpkg.json` есть в корне проекта?** Проверьте, что он лежит рядом с `CMakeLists.txt` и в нём указана зависимость `sfml` (см. Шаг 3).

3. **Путь к встроенному vcpkg в `CMakePresets.json` верен?** Откройте `CMakePresets.json` и проверьте в секции `vs2026-x64` значение `CMAKE_TOOLCHAIN_FILE`. По умолчанию это:
   ```
   C:/Program Files/Microsoft Visual Studio/18/Community/VC/vcpkg/scripts/buildsystems/vcpkg.cmake
   ```
   Если у вас установлена Professional / Enterprise версия или VS лежит на другом диске — поправьте этот путь. Узнать фактический путь можно в Developer PowerShell:
   ```powershell
   where.exe vcpkg
   ```
   замените `vcpkg.exe` в конце на `scripts/buildsystems/vcpkg.cmake`.

4. **После всех изменений** — в Visual Studio: **Проект → Удалить кэш и перенастроить** (Project → Delete Cache and Reconfigure), или вручную удалите папки `build/` и `.vs/` и откройте папку в VS заново.

5. В окне «Вывод» вы должны увидеть строки «Installing sfml...» и дождаться их окончания (10–30 минут при первой настройке).

### «error: Could not locate a manifest (vcpkg.json) above the current working directory» при ручном вызове `vcpkg install ...`
- Это нормально: встроенный в Visual Studio 2026 vcpkg работает **только в manifest-режиме** и не поддерживает команды вида `vcpkg install sfml:x64-windows`. Сборка SFML происходит автоматически во время настройки CMake в VS — ручных команд выполнять не нужно.

### «Generator: Visual Studio 18 2026 could not find an instance of Visual Studio» / «Не удалось найти экземпляр Visual Studio»
- У вас установлена более старая версия Visual Studio (например, 2022), а пресет требует именно 2026. Установите Visual Studio 2026 (см. Шаг 1) или замените значения `generator` и `toolset` в `CMakePresets.json` обратно на `"Visual Studio 17 2022"` / `"v143"`.

### «CMake Error: Unknown generator: Visual Studio 18 2026»
- У вас слишком старая версия CMake — генератор `Visual Studio 18 2026` появился только в CMake 4.2. В Visual Studio 2026 встроен **CMake 4.1.2**, и этого уже достаточно (мы используем `cmake_minimum_required(3.25)`). Если же вы вызываете CMake из командной строки самостоятельно — обновите его до 4.2 или новее (страница загрузки: [cmake.org/download](https://cmake.org/download/)).

### «error LNK2019 / LNK2001 / unresolved external symbol» при линковке SFML
- vcpkg собрал SFML тулсетом, не совпадающим с вашей Visual Studio (например, SFML собран v143, а проект собирается v145). Часто бывает, если SFML устанавливали при наличии нескольких версий Visual Studio.
- Удалите SFML из vcpkg и переустановите:
  ```powershell
  cd C:\dev\vcpkg
  .\vcpkg.exe remove sfml:x64-windows
  .\vcpkg.exe install sfml:x64-windows
  ```
- В Visual Studio: **Проект → Удалить кэш и перенастроить**.

### «vcpkg integrate install» выдаёт ошибку прав доступа
- Запустите PowerShell от имени администратора и повторите команду.

### Игра запускается, но окно сразу закрывается
- Скорее всего, не найден шрифт. Убедитесь, что рядом с `Snake.exe` есть папка `assets/` с файлом `DejaVuSans.ttf`. Если её нет — выполните сборку ещё раз (CMake копирует ассеты после сборки автоматически).

### Кириллица в окне отображается «квадратиками»
- Используйте только шрифт `DejaVuSans.ttf` из папки `assets/`. Если вы его удалили — восстановите файл из архива проекта.

### Visual Studio не показывает пресет vs2026-x64
- Закройте папку проекта (**Файл → Закрыть папку**) и откройте заново.
- Проверьте, что в установленных компонентах Visual Studio 2026 есть «Инструменты C++ CMake для Windows» (см. Шаг 1).

### Хочу собрать вручную из командной строки
Откройте «Developer PowerShell for VS 2026» (Пуск → Visual Studio 2026 → Developer PowerShell for VS 2026) и в папке проекта выполните:
```powershell
cmake --preset vs2026-x64
cmake --build build/vs2026-x64 --config Release
```
Готовый бинарник — `build\vs2026-x64\bin\Release\Snake.exe`.
